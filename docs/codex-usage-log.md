# Codex 사용 기록

## 작성 규칙

- 이 문서는 사용자가 Codex에 어떤 유형의 작업을 지시했는지 기록한다.
- 작업 결과나 상세 구현 내용이 아니라 지시의 성격을 중심으로 남긴다.
- 날짜 기준으로 나누지 않고 전체 사용 목적을 누적 관리한다.
- 각 항목은 `## 사용 목적` 형식으로 작성한다.
- 이미 기록된 사용 목적은 중복해서 다시 추가하지 않는다.

## 운영 원칙

- 앞으로 사용자가 Codex에 새로운 유형의 작업을 지시할 때마다 이 문서에 추가한다.
- 이미 존재하는 사용 목적이면 다시 기록하지 않는다.
- 새로운 지시를 처리할 때 현재 요청의 성격이 기존 항목에 없는지 먼저 확인한 뒤, 없으면 바로 갱신한다.

## 문서 작성

- 저장소 가이드, 프로젝트 개요, 도메인 설계, ERD, 개발 규칙, 개발 로그 같은 문서 작성 작업에 사용함

## 문서 수정

- 기존 문서의 구조 조정, 표현 수정, 섹션 분리, 파일명 변경 같은 문서 보정 작업에 사용함

## 구조 설계

- 도메인 구조, 인증/인가 방향, 계층 분리 방식 같은 설계 판단 정리에 사용함

## 작업 기록 관리

- Codex 사용 기록 방식 정의, 운영 규칙 추가, 중복 기록 방지 기준 정리 같은 기록 관리 작업에 사용함

## 스킬 생성

- 반복 작업 자동화를 위한 Codex 스킬 설계, 생성, 사용 방법 문서화 작업에 사용함

## 스킬 수정

- 기존 Codex 스킬의 이름, 동작 규칙, 적용 범위, 보조 문서 연동 방식을 수정하는 작업에 사용함

## 커밋 작업

- 현재 변경사항을 기준으로 커밋 타입을 정하고, 헤더와 본문을 구성해 git 커밋을 수행하는 작업에 사용함
